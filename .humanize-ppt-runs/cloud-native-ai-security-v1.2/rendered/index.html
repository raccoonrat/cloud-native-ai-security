The Shift

Enterprise AI moved from text generation to tool execution

MCP and tool-calling expose the first real security requirements when AI enters the operations control plane — not a product-local problem, but a platform-wide one.

  • Natural language intent → API calls with side effects
  • Prompt injection can alter tool selection and arguments
  • Northbound API security is necessary but not sufficient

Runtime Flow

Every request follows the same path

User/App → Ingress Gateway → Context Builder → Orchestrator →
Parallel Detectors → Signal Fusion → Policy Runtime → Decision Engine →
Enforcement → Response → Trace + Evidence → Eval → Release Gate

  • Context-aware: same prompt, different risk by tenant / role / data class
  • Contract-based: every decision carries reason_code + matched_rules
10 / 20
Canonical Contract

One field for runtime stage: stage

request_type is deprecated — never use for policy, trace, or eval matching.

input_precheck rag_query_precheck rag_retrieved_context_check tool_pre_execution tool_post_execution agent_plan_check agent_step_check model_output_check batch_eval

CI schema tests fail on any non-enum stage value.

Decision Contract

Actions are a state machine — not a priority list

allow block safe_complete

terminal_actions

redact rewrite remove_chunk reduce_rank

transform_actions

require_approval escalate warn log_only

Stage × Action Matrix

  • Not every action is valid at every stage
  • require_approval is pending — not terminal
  • Disruptive ops need explicit_user_confirmation
  • Must carry approval_ticket_id or workflow_ref

MCP Boundary

MCP is an AI-mediated execution boundary

  • Natural language → API call translation introduces new attack surface
  • Same API operation, different risk by user / tenant / scope
  • All tool-mediated ops must pass through control plane at tool_pre_execution
  • Generic API wrapper tools default block unless policy explicitly allows

Design Principle

Instruction / Data Separation

Untrusted Data

  • User prompts
  • RAG retrieved chunks
  • External web content
  • Tool output from third parties

Trusted Instructions

  • System prompts (signed, versioned)
  • Policy rules (immutable snapshot)
  • Approved tool schemas
  • Explicit user confirmations

External content must never become executable instructions in the agent loop.

18 / 20
Evidence

Two-phase trace: sync commit + async enrichment

Minimal Sync Commit

  • trace_id on every decision — no exceptions
  • reason_code on every non-allow
  • policy_snapshot_id + stage + action
  • Blocks "decision without trace" contradiction

Async Evidence

  • Full detector outputs
  • Matched rules detail
  • Redaction diffs
  • Replay + audit + compliance

Replay key: content hash + context hash + detector versions + policy snapshot + engine versions
